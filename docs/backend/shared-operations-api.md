# Shared Operations Task API Documentation

## Overview
The Shared Operations Task API supports the lifecycle of tasks managed in the "共享运营管理" workspace that lives alongside the 财务云. The API enables portal clients to browse shared tasks, create or update entries, manage lifecycle transitions, capture discussions, and attach supporting evidence required by operations governance.

All endpoints are namespaced under `/api/shared-cloud` and are served over HTTPS. Requests must include an `Authorization: Bearer <token>` header representing an authenticated matrix platform user. Unless otherwise noted, requests and responses are encoded as JSON objects with UTF-8 encoding.

## Resource Model
### Task Object
| Field | Type | Description |
| --- | --- | --- |
| `id` | string (UUID) | Unique identifier generated by the server. |
| `title` | string | Required. Task title with the "模块-场景-关键词" naming convention. Max 120 characters. |
| `description` | string | Required Markdown content describing scenario, expectation, and validation method. Minimum 20 characters. |
| `priority` | enum (`P0`, `P1`, `P2`, `P3`) | Required. Controls SLA defaults and priority badge coloring. |
| `slaClass` | enum (`SLA-P0`, `SLA-P1`, `SLA-P2`) | Auto-derived from `priority` but can be overridden by clients with `overrideSlaClass=true`. |
| `requester` | object | Non-editable user summary of the creator (`{ id, name, email, avatar }`). |
| `assignee` | object \| null | Assigned owner (`{ id, name, email }`). Nullable until pull-based claim occurs. |
| `status` | enum (`待分诊`, `待办`, `进行中`, `待验收`, `已完成`, `搁置`, `阻塞`) | Required. Timestamped transitions recorded in `eventLog`. |
| `dueDate` | string (ISO 8601 date) \| null | Optional due date. Must be on or after `createdAt`. |
| `labels` | string[] | Up to 3 tags. Duplicates rejected. |
| `module` | string | Required functional module identifier, e.g., `支付`, `账户`. |
| `acceptanceCriteria` | string[] | At least one verifiable acceptance statement. |
| `impactScope` | enum (`单用户`, `≤1%`, `1–5%`, `>5%`, `关键客户`, `金额区间`) \| string | Optional impact segment used for reporting. |
| `ticketSource` | enum (`客户`, `运营`, `监控`, `内部`, `渠道名`) \| object | Optional origin descriptor. The object format is `{ type: enum, detail?: string }`. |
| `taskType` | enum (`缺陷`, `需求`, `运营`, `支持`, `数据`, `自动化`) | Required. Drives dynamic form sections on the client. |
| `estimate` | object | Optional sizing: `{ value: number, unit: 'h' | 'pts' }`. `value` must be ≥ 0 with one decimal precision. |
| `subtasks` | array | For complex work. Array of `{ id, title, done }`. Requires at least 1 entry when provided. |
| `dependencies` | string[] | Optional list of blocking task IDs or external references. |
| `milestone` | string \| null | Optional release milestone identifier. |
| `quarter` | string \| null | Optional fiscal quarter label, e.g., `FY25Q4`. |
| `impactScopeDetail` | string \| null | Optional free-form impact description when enum is insufficient. |
| `ownerTeam` | string | Required owning team when `taskType` is not `缺陷`. |
| `coOwners` | user[] | Optional collaborators. Each item mirrors requester structure. |
| `env` | enum (`生产`, `预发`, `测试`) | Required when `taskType` is `变更类` (treated as `需求` or `自动化`). |
| `rollbackPlan` | string \| null | Required for change-type tasks. Markdown supported. |
| `testEvidence` | Attachment[] | List of uploaded files or URLs verifying validation. Each entry is `{ id, type: 'file' | 'url', name, url }`. |
| `attachments` | Attachment[] | Supporting documents attached to the task. |
| `relatedLinks` | string[] | Array of URLs (PRs, dashboards, docs). Each item validated via URL regex. |
| `comments` | CommentThread | Threaded discussion. Not returned in list endpoints unless `include=comments` is set. |
| `eventLog` | Event[] | System generated timeline of state changes and SLA events. |
| `firstResponseAt` | string (ISO 8601 datetime) \| null | Auto-populated on first transition away from `待分诊`. |
| `createdAt` | string (ISO 8601 datetime) | Creation timestamp. |
| `completedAt` | string (ISO 8601 datetime) \| null | Recorded when status becomes `已完成`. |
| `ageDays` | number | Computed property `(today - createdAt)` in days. Highlighted when > 7. |
| `riskLevel` | enum (`高`, `中`, `低`) | Required. `高` triggers extra confirmation on updates. |
| `dorCheck` | ChecklistItem[] | Entry criteria. Each item is `{ id, title, done, description? }`. At least one item must be present. |
| `rollbackPlanRequired` | boolean | Server derived flag telling the client whether `rollbackPlan` is mandatory. |
| `ciStatus` | enum (`通过`, `失败`, `进行中`, `未知`) | Read-only CI integration status. |
| `expectedValue` | string \| number \| null | Optional expected business outcome. Strings may contain Markdown. |
| `postmortem` | string \| null | Optional URL linking to retrospective once completed. |
| `csat` | number \| null | Optional acceptance satisfaction rating (1–5). |
| `duplicateOf` | string \| null | Optional pointer to canonical task ID. When set, server auto-sets status to `已完成` with subtype `重复`. |
| `reopenCount` | number | Read-only count of reopen cycles. |
| `securityLevel` | enum (`公开`, `内部`, `涉密`) | Controls visibility. `涉密` tasks require elevated scope. |

### CommentThread Object
| Field | Type | Description |

| `id` | string | Thread identifier per task. |
| `total` | number | Total comments in the thread. |

| --- | --- | --- |

| `mentions` | user[] | Mentioned users parsed from `@` references. |
| `createdAt` | string | Timestamp. |
| `attachments` | Attachment[] | Optional inline attachments. |

| Field | Type | Description |
| --- | --- | --- |
| `type` | enum (`STATUS_CHANGED`, `ASSIGNEE_CHANGED`, `LABELS_UPDATED`, `SLA_BREACH`, `COMMENT_ADDED`, `FILE_UPLOADED`, `ROLLBACK_PLAN_UPDATED`, `DUPLICATE_MARKED`) | Event classifier. |
| `from` | any | Previous value (when applicable). |
| `to` | any | New value (when applicable). |

### Attachment Object
| Field | Type | Description |

## Authentication & Permissions
* OAuth2 bearer tokens are validated by the platform gateway.
* `shared-operations:reader` grants read-only access to tasks and reference data.
* `shared-operations:editor` enables create/update of tasks, comments, and attachments.
* `shared-operations:admin` allows force transitions (e.g., override status, delete attachments), manage SLA overrides, and mark duplicates.
* Tasks with `securityLevel = '涉密'` additionally require `scope:shared-operations:confidential`.

### 2. Task Collection
#### 2.1 List tasks
`GET /api/shared-cloud/tasks`

Query parameters:
* `status`: filter by status enum.
* `module`: filter by module id.
* `priority`: filter by priority.
* `assignee`: filter by user id.
* `labels`: comma-delimited tags.
* `search`: fuzzy search on title/description.
* `page`: page index (default 1).
* `pageSize`: items per page (default 20, max 100).
* `sort`: `createdAt`, `dueDate`, or `priority`. Prefix with `-` for descending.

Response
```json
{
  "data": [
    {
      "id": "task_001",
      "title": "支付-退款失败-重复扣款",
      "priority": "P1",
      "slaClass": "SLA-P1",
      "status": "待验收",
      "module": "payment",
      "dueDate": "2024-08-12",
      "labels": ["支付", "退款"],
      "requester": { "id": "u_001", "name": "李雷" },
      "assignee": { "id": "u_101", "name": "韩梅" },
      "ageDays": 3,
      "riskLevel": "中"
    }
  ],
  "pagination": { "page": 1, "pageSize": 20, "total": 67 }
}
```

#### 2.2 Create task
`POST /api/shared-cloud/tasks`

Headers: `Idempotency-Key` recommended to guarantee single creation per request.

Request body
```json
{
  "title": "支付-退款失败-重复扣款",
  "description": "详细描述复现场景...",
  "priority": "P1",
  "taskType": "缺陷",
  "module": "payment",
  "status": "待分诊",
  "acceptanceCriteria": ["输入 → 复现", "处理 → 观察退款触发", "输出 → 金额校验"],
  "impactScope": "≤1%",
  "riskLevel": "中",
  "labels": ["支付", "退款"],
  "estimate": { "value": 12, "unit": "h" },
  "dependencies": ["FIN-1234"],
  "ownerTeam": "共享运营-支付保障",
  "env": "生产",
  "rollbackPlan": "若变更失败，回滚退款服务版本至v1.8",
  "testEvidence": [{ "id": "file_889", "type": "file" }],
  "attachments": [{ "id": "file_889", "type": "file" }],
  "relatedLinks": ["https://git.example.com/pr/42"],
  "dorCheck": [
    { "id": "dor1", "title": "场景描述清晰", "done": true },
    { "id": "dor2", "title": "风险识别完成", "done": true }
  ]
}
```

Validation rules:
* `title`, `description`, `priority`, `taskType`, `module`, `status`, `riskLevel` required.
* `description` must be ≥ 20 characters after Markdown stripping.
* `labels` limited to three unique entries.
* `dueDate` (if provided) must be ≥ `createdAt`.
* `rollbackPlan` mandatory when `taskType` in (`需求`, `自动化`) AND `env` is `生产` or `预发`.

Response `201 Created`
```json
{
  "data": {
    "id": "task_001",
    "createdAt": "2024-08-10T02:12:45Z",
    "status": "待分诊",
    "slaClass": "SLA-P1",
    "firstResponseAt": null
  }
}
```

#### 2.3 Get task detail
`GET /api/shared-cloud/tasks/{id}`

Query options:
* `include=comments,attachments,eventLog` - comma-separated expansions.
* `maskConfidential=true` - hides `description`, `rollbackPlan`, and `attachments` unless requester has confidentiality scope.

Response includes complete Task object. If `duplicateOf` is present, `duplicateTarget` will be embedded with `{ id, title, status }`.

#### 2.4 Update task
`PATCH /api/shared-cloud/tasks/{id}`

Supports JSON Merge Patch semantics. Server enforces state machine rules and records events. `status` transitions require `statusTransitionNote` field describing rationale.

Example
```json
{
  "status": "进行中",
  "assignee": { "id": "u_101" },
  "statusTransitionNote": "负责人已认领，排期处理中",
  "acceptanceCriteria": ["输入已复现", "处理监控报警修复", "输出退款成功"]
}
```

Special behaviors:
* Setting `assignee` to `null` reopens the task to the pool.
* When `status` becomes `已完成`, `completedAt` auto-populates and `csatPending` flag triggers acceptance workflow.
* Changing `riskLevel` to `高` requires `admin` scope and responds with `409` if missing.

#### 2.5 Delete task (admin only)
`DELETE /api/shared-cloud/tasks/{id}`

Soft deletes the task (`status = '删除'`) and appends an event log entry. Requires `shared-operations:admin`.

### 7. Webhooks & Integrations
#### 7.1 Task lifecycle webhook
Configure webhook subscriptions via `POST /api/shared-cloud/integrations/webhooks` with event types:
* `task.created`
* `task.updated`
* `task.status_changed`
* `task.completed`
* `task.duplicate_marked`

Webhook payload example:
```json
{
  "id": "evt_901",
  "type": "task.status_changed",
  "occurredAt": "2024-08-11T09:02:00Z",
  "data": {
    "taskId": "task_001",
    "fromStatus": "进行中",
    "toStatus": "待验收",
    "actor": { "id": "u_101", "name": "韩梅" }
  }
}
```

Webhook delivery uses HMAC-SHA256 signatures in the `X-Matrix-Signature` header. Clients must respond `2xx` within 10 seconds to avoid retries.

## Error Handling
Errors follow RFC 7807 (Problem Details) structure.

```json
{
  "type": "https://docs.matrix.io/errors/validation",
  "title": "Validation Failed",
  "status": 422,
  "traceId": "abc123",
  "errors": {
    "title": ["标题必填且不超过120字符"],
    "labels[2]": ["最多 3 个标签"]
  }
}
```

Common status codes:
* `400` – malformed request payload.
* `401` – missing/invalid token.
* `403` – insufficient scope or confidential task without clearance.
* `404` – resource not found or filtered by security policy.
* `409` – conflicting update, e.g., SLA override without permission or stale version.
* `410` – task marked duplicate; includes `duplicateOf` pointer.
* `422` – validation errors.
* `429` – rate limit exceeded (max 60 write ops/min per user).
* `500` – unexpected server error.

## Rate Limiting
* `GET` endpoints: 120 requests/minute per token.
* Write endpoints: 60 requests/minute per token.
* Webhook deliveries retried up to 5 times with exponential backoff.

## Audit & Observability
* Every mutating call logs `traceId`, user, IP, and payload diff to the audit store.
* `eventLog` entries are persisted for 365 days and exposed to audit reviewers.
* Metrics are exported via Prometheus under `shared_cloud_*` prefixes (e.g., `shared_cloud_task_created_total`).

## Change Management
* Versioning uses semantic versioned base path (`/api/shared-cloud/v1/...`). All endpoints above assume `v1`.
* Breaking changes require publishing deprecation headers (`Sunset`, `Deprecation`). Clients are notified through webhook `api.deprecation` events.

## Appendix A – Task Status Flow
```
待分诊 → 待办 → 进行中 → 待验收 → 已完成
             ↘ 搁置 ↗         ↘ 阻塞 ↗
```
* `搁置` requires `resumeAt` field when moving back to `待办`.
* `阻塞` requires specifying `blockedBy` dependencies to surface in UI.

## Appendix B – SLA Defaults
| Priority | First Response (h) | Resolution (h) |
| --- | --- | --- |
| `P0` | 1 | 8 |
| `P1` | 4 | 24 |
| `P2` | 8 | 72 |
| `P3` | 24 | 120 |

These defaults auto-populate `slaClass` unless an explicit override is provided.
